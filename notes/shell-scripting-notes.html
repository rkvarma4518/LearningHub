Shell Basics

A shell script is just a text file full of the same commands you'd type into a terminal, saved so you can run them all at once. This page covers the shebang, running scripts, variables, and getting input/output.

bash Variables stdin/stdout Arguments
01

What Is a Shell Script?

A shell script automates a sequence of terminal commands. Instead of typing commands one by one, you write them in a file and run the file. Most scripts on Linux/macOS use bash.

hello.sh
#!/bin/bash
# This is a comment
echo "Hello, World!"

The first line #!/bin/bash is called the shebang — it tells the system which program should run the file.

02

Running a Script

terminal
chmod +x hello.sh     # make it executable, do this once
./hello.sh              # run it

# or, without making it executable:
bash hello.sh
💡
chmod +x gives the file "execute" permission. Without it, ./hello.sh will say "permission denied".
03

Variables

No spaces around =. To read a variable's value, put a $ in front of its name.

variables.sh
name="Alex"
age=25

echo "My name is $name"
echo "My name is ${name}"   # same thing, braces are safer

# command output stored in a variable
today=$(date +%Y-%m-%d)
echo "Today is $today"
⚠️
name = "Alex" (with spaces) will not work — bash reads it as running a command called name with arguments.
04

Output & Input

greet.sh
echo "What is your name?"
read name
echo "Nice to meet you, $name!"

# read with a prompt on one line
read -p "Enter your age: " age
echo "You are $age years old"
05

Command-Line Arguments

Values passed when running the script — e.g. ./deploy.sh prod v2.

SymbolMeaning
$0The script's own name
$1, $2, …First, second, … argument
$#Number of arguments passed
$@All arguments, as separate words
$?Exit code of the last command run
deploy.sh
echo "Script name: $0"
echo "Environment: $1"
echo "Version: $2"
echo "Total args: $#"
06

Quoting Basics

QuoteBehavior
"double"Variables and $(...) are expanded inside
'single'Everything is literal — nothing is expanded
no quotesWord splitting happens — risky with spaces in values
💡
A good habit: always wrap variables in double quotes, like "$name", to avoid surprises if the value contains spaces.
Page 02 / Logic

Control Flow

Making decisions and repeating work — if/else, case, the three loop types, and reusable functions.

if / else for / while case functions
01

if / elif / else

Square brackets [ ] are the test command. Always leave a space right after [ and before ].

check_age.sh
if [ $age -ge 18 ]; then
  echo "You are an adult"
elif [ $age -ge 13 ]; then
  echo "You are a teenager"
else
  echo "You are a child"
fi

Comparison operators

NumbersStringsMeaning
-eq=equal
-ne!=not equal
-gt—greater than
-lt—less than
-ge / -le—greater/less than or equal
—-zstring is empty
—-nstring is not empty
📘
Combine conditions with && (and) or || (or): if [ "$a" = "x" ] && [ "$b" = "y" ]; then
02

case

A cleaner way to check one variable against many possible values.

case.sh
read -p "Enter a fruit: " fruit

case $fruit in
  apple)
    echo "That's a red fruit" ;;
  banana)
    echo "That's a yellow fruit" ;;
  grape|orange)
    echo "That's a common fruit" ;;
  *)
    echo "Unknown fruit" ;;
esac
03

Loops

for loop
for i in 1 2 3; do
  echo "Number: $i"
done

# loop over a range
for i in {1..5}; do echo $i; done

# loop over files
for file in *.txt; do
  echo "Found: $file"
done
while / until loop
count=1
while [ $count -le 5 ]; do
  echo "Count: $count"
  count=$((count + 1))
done

# until runs while the condition is FALSE
until [ $count -gt 10 ]; do
  echo $count
  count=$((count + 1))
done
💡
break exits a loop early; continue skips to the next iteration — same as most other languages.
04

Functions

Functions group reusable code. Arguments are read with $1, $2, etc, just like script arguments.

functions.sh
greet() {
  local name=$1
  echo "Hello, $name!"
}

add() {
  local sum=$(($1 + $2))
  echo $sum   # "return" a value by printing it
}

greet "Sam"
result=$(add 3 4)
echo "Sum is $result"
📘
local keeps a variable scoped to the function, so it doesn't overwrite a same-named variable outside it.

Files & Text

Redirection Pipes Arrays grep / sed / awk
01

File Test Operators

Used inside [ ] to check things about a file before acting on it.

OperatorChecks
-eFile or directory exists
-fExists and is a regular file
-dExists and is a directory
-r / -w / -xReadable / writable / executable
-sFile exists and is not empty
check_file.sh
if [ -f "config.txt" ]; then
  echo "Config file found"
else
  echo "Missing config.txt"
fi
02

Redirection & Pipes

SymbolMeaning
>Send output to a file, overwriting it
>>Send output to a file, appending to it
<Read input from a file
2>Send only error output to a file
|Pipe — send one command's output into the next command's input
examples
echo "log entry" >> app.log     # append to a log file
ls -l > files.txt             # save listing to a file
command 2> errors.txt        # save only errors
ps aux | grep "nginx"       # pipe: filter process list
cat file.txt | wc -l          # pipe: count lines
03

Arrays

arrays.sh
fruits=("apple" "banana" "grape")

echo ${fruits[0]}        # first item: apple
echo ${fruits[@]}        # all items
echo ${#fruits[@]}       # number of items

for fruit in "${fruits[@]}"; do
  echo "I like $fruit"
done
04

grep — Search Text

grep examples
grep "error" app.log         # lines containing "error"
grep -i "error" app.log      # case-insensitive
grep -v "debug" app.log      # lines NOT containing "debug"
grep -r "TODO" ./src         # search recursively in a folder
grep -c "error" app.log      # just count matching lines
05

sed — Find & Replace

sed examples
sed 's/old/new/' file.txt      # replace first match per line, prints result
sed 's/old/new/g' file.txt     # replace ALL matches per line
sed -i 's/old/new/g' file.txt  # edit the file in place
sed -n '2,4p' file.txt        # print only lines 2 to 4
06

awk — Columns & Reports

Great for column-based data — awk splits each line into fields ($1, $2, …) by whitespace by default.

awk examples
awk '{print $1}' file.txt        # print the first column of each line
awk -F"," '{print $2}' data.csv   # use comma as the separator
awk '{sum += $3} END {print sum}' data.txt  # total up column 3
💡
Rule of thumb: grep to find lines, sed to change text, awk to work with columns.

01

Exit Codes

Every command returns a number when it finishes: 0 means success, anything else means some kind of failure. Stored in $?.

exit_codes.sh
mkdir /tmp/testdir
if [ $? -eq 0 ]; then
  echo "Folder created successfully"
else
  echo "Something went wrong"
fi

# exit your own script with a custom code
exit 1
02

String Tricks

strings.sh
text="Hello World"

echo ${#text}              # length: 11
echo ${text:0:5}          # substring: Hello
echo ${text/World/Bash}   # replace: Hello Bash
echo ${text^^}             # uppercase: HELLO WORLD
echo ${text,,}             # lowercase: hello world
03

Debugging a Script

terminal
bash -x script.sh      # print every command as it runs
bash -n script.sh      # check for syntax errors without running it

You can also add set -x near the top of a script to turn on this tracing just for that script, and set -e to make it stop immediately if any command fails.

⚠️
Adding set -euo pipefail at the top of a script is a common safety habit: stop on error, stop on unset variables, and catch failures inside pipes.
04

Scheduling With cron

cron runs a script automatically on a schedule. Edit your schedule with crontab -e.

crontab syntax
# minute hour day month weekday   command
0 9 * * *   /home/user/backup.sh        # every day at 9:00 AM
*/15 * * * * /home/user/check.sh        # every 15 minutes
0 0 * * 0    /home/user/weekly.sh        # every Sunday at midnight
